===================================================================
AWSクラウドプラクティショナー学習アプリ 構想メモ
===================================================================
【1. アプリ概要】
　
　・名称(仮)：AWSクラプラ攻略アプリ(仮称)
　・目的　　：AWS Certified Cloud Practitioner 合格をサポートする学習アプリ
　・対象　　：これからクラプラを受ける初学者～ライトエンジニア
　・料金　　：基本無料 ＋ 一部機能を 500円 の買い切り課金で開放
　・プラットフォーム：まずは Android (Google Play) からスタート
　・開発体制：自分 ＋ ChatGPT(相棒) で少しずつ作る前提

-------------------------------------------------------------------
【2. 課金イメージ】

　・課金方式：
　　→ アプリ内課金 (買い切り / 一回 500円)

　・課金で開放されるもの (まとめて一括解放)：
　　→ 基礎講義：第6～13章
　　→ 練習問題：第6～13章の問題
　　→ 模擬試験：第3回以降の模試チャレンジ権（全15章：1章30問）

　・無料で使える範囲：
　　→ 基礎講義：第1～5章
　　→ 練習問題：第1～5章
　　→ 模擬試験：2回分まで
　　→ チェック：制限なし

-------------------------------------------------------------------
【3. 機能仕様 (ざっくり)】

　　(1) 基礎講義モード (1～13章)
　　　・構成：全13章 (クラプラの範囲を自作テキストで整理)
　　　・第1～5章：無料で閲覧可能
　　　・第6～13章：課金後に解放
　　　・各章には「要点まとめ」「図解イメージ」「用語集」などを掲載する想定
　　
　　(2) 練習問題モード (章別)
　　　・各章ごとに選択式の問題を用意
　　　・1問ずつ出題し、解答後に正誤と解説を表示
　　　・第1～5章の問題：無料
　　　・第6～13章の問題：課金で解放 (講義とセット)
　　
　　(3) 模擬試験モード
　　　・クラプラ本番を意識した構成
　　　　・1セット：65問前後
　　　　・本番と同じように時間を計測
　　　・模試セットを10個くらい用意するイメージ
　　　・無料で挑戦できるのは2回分 (例：模試1・模試2)
　　　・3回目以降の模試は課金したユーザーのみ挑戦可能
　　　・終了後にスコアと分野別の正答率を表示
　　
　　(4) 苦手克服モード
　　　・機能：
　　　　・間違えた問題を自動で保存
　　　　・ユーザーが「しおり」マークを付けた問題も保存
　　　・「苦手問題だけを出題」「しおり問題だけを出題」などのモードを用意
　　　・ここは課金制限なし (無料ユーザーもフルで使える想定)
　　
　　(5) ユーザーデータ管理 (ローカル)
　　　・スマホ内に以下を保存 (サーバ無しでも動く構成にする)
　　　　・既に正解したかどうか
　　　　・間違えた回数
　　　　・しおりフラグ
　　　　・課金済みかどうかのフラグ
　　　・初期バージョンではオンライン認証は行わず「1端末完結」でOK

【4. データ構造 (イメージ)】

・問題データは JSON などで持つ想定

例）1問のイメージ：

id　　　　： "Q001-01"
chapter　 ： 1
type　　 ： "single" (単一選択)
question ： "クラウドコンピューティングの特徴として正しいものはどれか？"
choices　： ["A: ～", "B: ～", "C: ～", "D: ～"]
answerIndex： 2 (0始まりで正解の選択肢)
explanation： "クラウドの特徴は～"
domain　 ： "クラウド概念"
tags　　 ： ["基礎", "オンプレ比較"]

※後で相棒と一緒に実際の JSON フォーマットを決める。

【5. 画面イメージ (ざっくり)】

　(1) ホーム画面
　　・メニュー：
　　　テキスト
　　　練習問題
　　　模擬試験
　　　チェック問題
　　　設定
     test

　(2) 講義画面
　　・章一覧 (ロック状態は鍵アイコンで表示)
　　・章詳細画面でテキストと簡単な図解を表示
　　・画面下部に「この章の問題に進む」ボタン

　(3) 問題画面 (練習 / 模試共通)
　　・問題文と選択肢を表示
　　・回答後に「正解 / 不正解」「解説」「しおりボタン」
　　・「次の問題へ」ボタン

　(4) 苦手克服画面
　　・「間違えた問題から出題」「しおり問題から出題」を選択できる
　　・対象問題をランダムまたは順番に出題

　(5) 課金・ロック解除画面
　　・500円課金で解放される内容を一覧表示
　　・購入状態 (購入済み / 未購入) を表示

【6. 守るべきルール (AWS / 試験まわり)】

　・本試験の問題文そのものの転記は禁止 (絶対NG)
　・必ずオリジナル問題として作成する
　・過去問サイトやブレインダンプ等から問題を流用・改変しない
　・アプリ内に注意書きを入れる：
　　「本アプリに掲載している問題および解説は、
　　AWS公式ドキュメント（ホワイトペーパー、試験ガイド等）を参考に
　　作者および生成AIが独自に作成したものであり、
　　AWS認定試験で実際に出題された問題を転記したものではありません。」
　・アプリ内に非公式であることを明記する：
　「本アプリは Amazon Web Services (AWS) による公式アプリではなく、
　　いかなる承認・後援・提携関係もありません。
　　『AWS』および関連名称は Amazon.com, Inc. またはその関連会社の商標です。」
　・あくまで「試験対策の自作教材」という位置づけにする

【7. 今後やること (ロードマップ)】

　Step 1：戦略と技術の決定
　　・まずは Android 版からスタート (Google Play)
　　・開発フレームワークを決める (Flutter か React Native)
　　・PC に開発環境を入れる (相棒に聞きながら進める)

　Step 2：ミニマム版のアプリを作る
　　・「1章だけ ＋ 問題3問だけ」のミニアプリを作る
　　・JSON の問題データを読み込んで表示する
　　・正解・不正解の判定と解説表示を実装する

　Step 3：機能を広げる
　　・章1～5までの講義と問題を実装する
　　・苦手克服モード (間違えた問題一覧) を実装する
　　・模擬試験モード (1セットだけ) を仮実装する

　Step 4：課金機能と本番公開準備
　　・Google Play デベロッパー登録 (25ドル)
　　・アプリ内課金で「300円の買い切り」を実装する
　　・無料 / 有料のロック制御を実装する
　　・アイコン・スクリーンショット・説明文を準備する
　　・テスト後、Google Play に申請・公開する

　Step 5：改善・拡張
　　・模擬試験セットを増やす
　　・解説や講義テキストをブラッシュアップする
　　・余裕があれば iOS 対応も検討する (Apple Developer 登録が必要)

【8. 問題作成ルールとプロンプト方針】

　(1) 問題作成の基本ルール
　　・AWS認定試験の実際の問題や、その模倣にならないようにする
　　・過去問・リーク問題・ブレインダンプを一切利用しない
　　・AWS公式ドキュメント（ホワイトペーパー、試験ガイド、FAQ 等）の内容を参考にしつつ、
　　　問題文・選択肢・数値・シナリオはすべてオリジナルで作成する
　　・同じ概念を問う場合でも、他教材と同一の文章・同一の数値にしない
　　・問題は ChatGPT(相棒) が生成し、自分が目視でチェックし、
　　　「どこかで見た本番問題そのものではないか」を確認してから採用する

　(2) 相棒に依頼するときの基本プロンプト例
　　
　　・共通テンプレ（安全プロンプト）
　　「AWS Certified Cloud Practitioner 向けの学習用オリジナル問題を作りたいです。
　　次の条件を必ず守ってください。
　　・AWS の認定試験で実際に出題された問題や、いわゆる過去問・ブレインダンプを一切使わないこと
　　・AWS 公式ドキュメント（ホワイトペーパー、試験ガイド、FAQ など）に書かれている内容を参考にしつつ、
　　　『新しく』『独自に』問題文と選択肢を作ること
　　・文章や数値、シナリオは他の教材と同一にならないように、毎回オリジナルな内容にすること
　　・クラウドプラクティショナーの出題範囲レベルで、初心者にも分かる難易度にすること

　　以上を守ったうえで、
　　『クラウドの基本概念（オンプレとの違い、スケーラビリティ、従量課金など）』に関する4択問題を5問作ってください。
　　各問について、
　　・question（問題文）
　　・choices（4択）
　　・correct_index（0～3）
　　・explanation（解説）
　　を JSON 形式で出力してください。」

・章ごとの指示イメージ

　例：第1章「クラウドの概要」

　　「クラウドの基本概念（オンプレとの違い、スケーラビリティ、従量課金など）を理解するための
　　オリジナル4択問題を5問作ってください。
　　条件は次の通りです：
　　・実際の AWS 認定試験問題や、その模倣にならないようにしてください
　　・AWS 公式のホワイトペーパーや試験ガイドに書かれている考え方をもとに、
　　　文章・数値・シナリオはすべて新しく作ってください
　　・クラウドプラクティショナーの範囲・難易度を想定してください
　　出力形式は前述の JSON 形式としてください。」

　(3) NGな依頼例（自分への注意メモ）
　　・「本番で出るような問題を再現して」
　　・「ネットにあるクラプラ過去問と同じ感じの問題を100問作って」
　　・「このURLの問題集を真似して、似た問題を作って」
　　　→ こういった指示は、本番問題やブレインダンプの模倣につながるため行わない。

【9. 開発難易度と自分の立ち位置メモ】

　(1) 難易度イメージ
　　・「まったくの初心者には重めだが、今の自分なら十分狙えるレベル」
　　・クラプラ学習や PowerShell バッチ、JSON 設計をこなしてきた経験があるため、
　　　一般的な「プログラミング未経験者」よりスタート地点はかなり有利
　　・ただし「1週間でサクッと終わるお遊び」ではなく、
　　　数ヶ月かけてコツコツ進める趣味プロジェクト寄りの難易度
　　
　(2) 難しく感じそうなポイント
　　・新しい言語／フレームワークを覚える必要がある
　　　（Flutter なら Dart、React Native なら JavaScript + React）
　　・画面（UI／UX）の考え方がサーバ系・バッチ系と違う
　　　（ボタン、画面遷移、状態管理など）
　　・リリース用ビルドとストア公開（証明書、ストア登録、スクショ準備など）の手順が多い
　　・一気に全部やろうとすると挫折しやすいので、
　　　小さく分割して進める工夫が必要
　　
　(3) 自分の強み
　　・PowerShell やバッチ処理、JSON 設計を仕事で使っており、
　　　ロジック構成やフロー設計には慣れている
　　・AWS クラプラの内容を理解しているため、
　　　「どんな機能や問題が必要か」を自分で決められる
　　・ログやエラー処理などをきちんと設計する習慣があるため、
　　　アプリの品質面でも強みになる
　　
　(4) 現実的な進め方イメージ
　　・まずは「画面1枚だけの Hello World アプリ」を相棒と一緒に作る
　　・次に「JSON で1問だけ問題を読み込んで表示」までをゴールにする
　　・その後「4択ボタン＋正解／不正解表示」「問題を複数に増やす」など、
　　　少しずつステップアップしていく
　　・最終的に、ロードマップ【7】の Step 1～5 を
　　　小さく分割しながらクリアしていくイメージで進める
